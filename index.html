Algoritma ve Programlamaya Giriş

Algoritma · Akış Diyagramı · Sözde Kod · Yazılım Geliştirme Süreci
Algoritma Nedir?

Adını El-Hârizmî'den alan algoritma; belirli bir problemi çözmek için izlenen sıralı, sonlu, açık ve mantıksal adımlar dizisidir.

  • Her adım net ve anlaşılır olmalı
  • İşlem sırası korunmalı
  • Tüm olası durumları kapsamalı
  • Başlangıç ve bitiş noktaları olmalı
  • Sonlu sayıda adım içermeli
Algoritmanın Önemi
  • Takım çalışmasını kolaylaştırır
  • Projeyi tasarlayan dışında kişi tarafından da kodlanabilir
  • Kodlama öncesi mantık hatalarını tespit eder
  • Günlük hayatın her alanında kullanılır
  • Karmaşık süreçleri basit adımlara indirger

Problem Çözme Aşamaları

1
Problemi Tanımlama (İnceleme, Analiz)

Problem ne kadar iyi anlaşılırsa algoritma o kadar kolay geliştirilir.

2
Girdi ve Çıktıların Belirlenmesi

Sistemin başlangıç ve bitiş noktaları, parametre giriş/çıkışları netleştirilir.

3
Alternatif Çözüm Yolları

Bir problemin birden fazla çözümü vardır; zamana ve kaynağa göre en uygun seçilir.

4
Çözümün Sınanması ve İyileştirilmesi

Kodlamadan önce algoritma adım adım test edilir, mantık hataları giderilir.

5
Algoritmanın Kodlanması

Uygun programlama dili seçilerek algoritma koda dönüştürülür.

6
Kodun Sınanması ve İyileştirilmesi

Belirlenen girdiler için doğru çıktılar alınıp alınmadığı test edilir.

Yazılım Geliştirme Süreci

1
Taleplerin Belirlenmesi

En kritik adım. Hatalı analiz tüm süreci etkiler. İş akış diyagramları ile desteklenir.

2
Planlama ve Analiz

Modüllerin detaylandırılması, dil/mimari seçimi. "Ne yapılacak" odaklı, "nasıl yapılacak" değil.

3
Tasarım

UI tasarımı, veri tabanı şemaları, UML diyagramları. Tüm teknik detaylar burada belirlenir.

4
Kodlama

Tasarım dokümanlarına göre yazılım geliştirilir. Kod yazım standartlarına uyulur.

5
Test ve Entegrasyon

Birim testler, sistem testleri ve son aşamada müşteri kabul testi yapılır.

6
Bakım

Operasyon sırasında oluşan hataların kalıcı çözümü. Proje büyüklüğüne göre şekillenir.

Derleyici (Compiler)

Tüm kaynak kodu makine diline çevirir. Hata varsa çalıştırmaz, tüm hataları bildirir. Hedef kod oluşturulurken optimizasyon yapılır (kullanılmayan değişkenler temizlenir).

Kaynak Kod → Derleyici → Hedef Kod
Yorumlayıcı (Interpreter)

Kodu satır satır veya blok blok çalıştırır. Çalışmayan bloktaki hata hiç görülmeyebilir. Derleme aşaması yoktur.

⚠️
Çalıştırılmayan bir kod bloğundaki hata asla görülmeyebilir!

Nesne Yönelimli Programlama — 4 Temel Özellik

Kapsülleme

Encapsulation

Sınıf özelliklerini ve metotlarını dış erişime karşı koruma altına almak. Erişim belirteçleri: public, private, protected

En güvensiz: public  |  En güvenli: private

Miras Alma

Inheritance

Alt sınıfın üst sınıfın tüm özellik ve metotlarını devralması. Var olan sınıf üzerine yeni sınıflar oluşturulabilir.

Örnek: Motorlu Taşıtlar → Araba, Motosiklet

Çok Biçimlilik

Polymorphism

poly=çok, morph=biçim. Türetilen sınıfın, kalıtımla gelen metot/özellikleri kendi ihtiyacına göre yeniden tanımlaması.

Örnek: Araba.hız ≠ Traktör.hız

Soyutlama

Abstraction

Üst soyut sınıfta, alt sınıflara şablon olarak metotlar tanımlanır. Soyut sınıftan nesne üretilemez.

abstract class → soyut sınıf

Programlama Dili Değerlendirme Ölçütleri

Yüzlerce programlama dili geliştirilmiştir. Bir dili diğerinden ayıran özellikleri ölçmek için şu ölçütler kullanılır:

ÖlçütAçıklama
Verimlilik (Efficiency)Programın hızlı ve az bellek kullanarak çalışması. Küçük kod → daha hızlı çalışma.
Veri Türleri ve YapılarıTam sayı, gerçek sayı, boolean, char gibi türleri; dizi, liste, ağaç gibi yapıları destekleme kapasitesi.
Alt Programlama Yeteneği (Modularity)Büyük problemi parçalara bölerek her parçayı ayrı fonksiyon/modülde çözebilme.
YapısallıkBloklar hâlinde yazım, alt programların yoğun kullanımı. Kodun okunurluğunu ve yönetimini kolaylaştırır.
Esneklik (Flexibility)Programcının donanıma geniş ölçüde erişebilmesi. Kısıtlama azaldıkça hata riski artar.
Öğrenme Kolaylığı (Pedagogy)Seviye yükseldikçe öğrenmesi kolaylaşır. C# öğrenmesi C'den çok daha kolaydır.
Genellik (Generality)Yalnızca belli işlemler için değil, çok çeşitli uygulamalar geliştirilebilmesi. Örn: C genel amaçlı hâle gelmiştir.
Giriş/Çıkış Kolaylığı (I/O Facility)Dosya erişimi ve veri tabanı işlemlerinin kolaylığı. MySQL, Oracle gibi dillerde bu özellik gelişmiştir.
Okunabilirlik (Readability)Kodun başkaları tarafından da kolayca anlaşılabilir yazılması. Ortak projelerde kritik öneme sahiptir.
Taşınabilirlik (Portability)Kaynak kodun farklı sistemlerde sorunsuz derlenip çalışabilmesi (standardizasyon). C dili bu açıdan gelişmiştir.
Nesne YönelimlilikNesne yapısı üzerine kurulu dil. Kodlar milyonlarca satıra ulaştığında kontrolü kolaylaştırır. C→C++, Pascal→Delphi.

Yapısal Programlama vs Nesne Yönelimli Programlama

Yapısal Programlama

1900'lü yılların ortasında, GOTO komutunun karmaşıklığına çözüm olarak doğmuştur.

  • Hiyerarşik, mantıksal akış yapılarından oluşur
  • Problem → küçük alt parçalara bölünür
  • Aynı alt program bloğu defalarca çağrılabilir
  • Kod tekrarını önler, okunurluğu yüksektir
  • if, döngüler ve fonksiyonlar temel yapı taşlarıdır
Nesne Yönelimli Programlama

1960'ların sonlarında, büyüyen kod tabanlarını yönetmek için geliştirilmiştir.

  • Gerçek dünya modellemesi yapar
  • Her şey bir nesnedir
  • Nesneler birbirleriyle iletişime geçerek program çalışır
  • Alan Kay tarafından önerilmiştir
  • Java, C++, Python, C#, PHP bu yaklaşımı destekler
💡
NYP'yi destekleyen dillerin tamamına yakını yapısal programlamayı da bünyesinde barındırır. Bu iki yaklaşım birbirini dışlamaz.

Çoklu Atama

Tek Satırda Çoklu Tanım

Aynı türde birden fazla değişken virgülle ayrılarak tek satırda tanımlanabilir.

// Tek satırda tanım + değer atama
int x = 5, y = 12, z;
// z'ye ilk değer atanmadı

// Önce tanımla, sonra ata
int a, b, c;
a = b = c = 7;
// a=7, b=7, c=7
Dikkat Edilecekler
  • Çoklu atama kodun anlaşılmasını zorlaştırır — dikkatli kullanılmalı
  • int x = y = 9; yazım hatasıdır — y daha önce tanımlanmamış
  • Atama işlemi sağdan sola yapılır: a = b = c = 7 → önce c=7, b=7, a=7
  • C#'ta kullanılmadan önce değer atanmamış değişken derleyici hatası verir

Geçerlik Bölgesi (Scope)

Yerel Değişken

Bir metot, döngü veya { } bloğu içinde tanımlanan değişkenlerdir. Yalnızca tanımlandıkları blokta geçerlidir. Blok tamamlanınca bellekten silinir.

for (int i = 0; i < 5; i++) {
  // i sadece burada geçerli
}
// i burada artık yok!
Sınıf Değişkeni

Sınıf içinde, metotların dışında tanımlanan değişkenlerdir. Sınıfın tüm metotlarından erişilebilir. Nesne bellekte kaldığı sürece yaşamaya devam eder.

class Ornek {
  int sinifDegiskeni = 0; // sınıf değişkeni
  void Metot1() { sinifDegiskeni++; }
  void Metot2() { sinifDegiskeni--; }
}

Değer Türleri vs Referans Türleri

Değer Türleri

Verinin kendisini taşırlar. Stack bellekte saklanır, hızlı erişim.

int · long · float · double · decimal · char · bool · byte · short · struct · enum

💡
Metoda gönderilince kopyalanır — orijinal değer değişmez.
Referans Türleri

Verinin adresini taşırlar. Heap bellekte saklanır.

string · object · class · interface · array · delegate · pointer

⚠️
İstisna: string referans türü olmasına rağmen metoda değer türü gibi kopyalanarak geçirilir.

ref ve out Anahtar Kelimeleri

ref — Referans ile geçirme

Değer türlerini metoda referansıyla gönderir. Metodun parametreyi değiştirmesine izin verir.

  • Metot bildirimi ve çağrımında her ikisinde de ref yazılmalı
  • Gönderilmeden önce değişkene değer atanmış olmalı
static void daireAlani(ref double d)
{
  d = 3.14 * d * d;
}
// Çağırma:
double r = 5;
daireAlani(ref r); // r artık alan değerini taşır
out — Birden fazla değer döndürme

Metodun birden fazla değer döndürmesini sağlar.

  • Gönderilen değişkene ilk değer atanma zorunluluğu yoktur
  • Değer, metot içinde hesaplanıp döndürülür
static int Max(int x, int y, out int kare)
{
  kare = (x > y) ? x*x : y*y;
  return Math.Max(x, y);
}
// Çağırma:
int kare;
int max = Max(9, 2, out kare);

Metotlar (Methods)

💡
Belirli görevler için tasarlanmış kod blokları. Mutlaka bir class veya struct içinde tanımlanmalıdır. Main() programın tek giriş noktasıdır.
// Metot şablonu:
public int UcgenAlani(int taban, int y) {
  int alan = (taban * y) / 2;
  return alan;
}

// void: dönüş değeri yok
static void EkranaYaz(string mesaj) {
  Console.WriteLine(mesaj);
}

// Main'den çağırma (static olmalı)
static void Main() {
  EkranaYaz("Merhaba!");
}
// Özyinelemeli (Recursive) Metot
static int Faktoriyel(int n) {
  if (n == 0) return 1;
  return n * Faktoriyel(n - 1);
}
// Faktoriyel(6) = 720
// 6×5×4×3×2×1×1 = 720
